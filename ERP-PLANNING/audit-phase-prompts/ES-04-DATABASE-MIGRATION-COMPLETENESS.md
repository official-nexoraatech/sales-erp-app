## ═══════════════════════════════════════════
## PROJECT CONTEXT
## ═══════════════════════════════════════════

### Tech Stack
Node.js 20 + TypeScript 5 | Fastify 4 | PostgreSQL 16 + Drizzle ORM | Turborepo + pnpm

### Migration Rules (CRITICAL)
- Migrations are **append-only** — NEVER edit an existing migration file
- Naming: `{next_sequence_number}_{description}.sql` (e.g., `0010_es04_missing_tables.sql`)
- Generated via `pnpm drizzle-kit generate` — do NOT hand-write SQL
- Must be idempotent: use `CREATE TABLE IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`
- REJECT any generated migration that contains `ALTER TABLE`, `DROP`, or `TRUNCATE` on existing tables
- All new tables must have: `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`, `tenant_id UUID NOT NULL`, `created_at TIMESTAMPTZ DEFAULT NOW()`, `updated_at TIMESTAMPTZ DEFAULT NOW()`
- All new tables must have index: `CREATE INDEX IF NOT EXISTS idx_{table}_tenant_id ON {table}(tenant_id)`

### Coding Standards
- No `console.log` — use `packages/logger`
- TypeScript strict mode
- Add `/* global process */` when using `process.env`

## OBJECTIVE
## ═══════════════════════════════════════════

Generate the missing migration SQL for all tables defined in `schema/production.ts` and `schema/report.ts` that have NO corresponding migration file. Verify on a clean PostgreSQL database.

**Why critical:** `job_work_orders`, `barcode_batches`, `barcodes`, `consignment_stocks`, `consignment_settlements`, `report_schedules`, and `report_run_history` are defined in schema but have NO migration. On any fresh production deployment, these tables DO NOT EXIST, causing immediate runtime crashes in `production-service` and `report-service`.


**Step 3: Review the generated SQL carefully**
- ACCEPT if: only `CREATE TABLE IF NOT EXISTS` and `CREATE INDEX IF NOT EXISTS`
- REJECT if: any `ALTER TABLE`, `DROP`, `TRUNCATE`, or changes to existing tables
- If drizzle-kit produces unexpected changes to existing tables, STOP and investigate before proceeding

**Step 4: Rename the generated file**
Name it: `{next_sequence}_{description}.sql`
Example: `0010_es04_production_report_tables.sql`
Add a header comment:
```sql

```

**Step 5: Verify on clean database**
```bash
# Start a fresh postgres:16 Docker container
docker run -d --name pg-test -e POSTGRES_PASSWORD=test -p 5433:5432 postgres:16

# Apply all migrations in sequence
pnpm drizzle-kit migrate --config=drizzle.config.ts

# Verify all 7 target tables exist
psql postgresql://postgres:test@localhost:5433/postgres -c \
  "SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' ORDER BY table_name;"
```

**Step 6: Start affected services against migrated DB**
